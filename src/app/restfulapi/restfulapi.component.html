A restful API megvalósítása mongoDB használatával

A mongoDB elindításához először parancssorban kiadom a net start mongoDB parancsot, majda a Robo 3T alkalmazáson keresztül, amely a mongoDB-hez ad grafikus felületet, létrehoztam egy új kapcsolatot messageService néven.

Ezután létrehoztam a messageService nevű adatbázist, amelyhez hozzáadtam a users és a messages adattáblákat (collection).

A messageService adatbázishoz új usert adok hozzá a szükséges paraméterekkel: felhasználónév, jelszó, jogosultságo. Végül beállítom az autentikációt, amely lehetővé teszi az adatbázishoz való hozzáférést. Parancssoron elindítom

Következő lépés a restful API létrehozása Visual Studio Code-ban. Elöször git inittel lokális repot hozok létre, majd git ignore paranccsal beállíttom, hogy a Git milyen fájlokat hagyjon figyelmen kívül.

npm instal paranccsal letöltöm az alkalmazáshoz szükséges package-okat, mint express, body-parser, morgan, helmet, mongoose, passport, majd létrehozom a package.json fájlt az npm init pranccsal. Ez a fájl tartalmazza a projektem legfontosabb adatait.


Ezután felépítem az alkalmazás mappa és fájl struktúrájá:

  • config mappában létrehozom a database.js fájlt
  • controllers mappában létrehozom a user.controller.js fájlt, amely az adatmodel vezérléséért felelős kódokat tartalmazza.
  • models mappában létrehozom a user.model.js fájlt, amelyben az adatmodel séma kerül definiálásra
  • routes mappában létrehozom a message.route.js fájlt, amely fájl a különböző útvonalak vezérléséért felel.
  • A server.js fájlban pedig többek között olyan metódusok kerülnek definiálásra, mint a csatlakozás a mongoDB-hez, beérkező kérések adatcsomagjainak összefűzése, kérések naplózása, alap security beállítások. HTTP szerver indítása.

A weboldal elkészítése Angularral

Első lépésként legeneráltam egy új angular projectet a ng new messagingService néven. Elöször git inittel lokális repot hozok létre, majd git ignore paranccsal beállíttom, hogy a git milyen fájlokat hagyjon figyelmen kívül.

npm instal paranccsal letöltöm az alkalmazáshoz szükséges package-okat, mint express, body-parser, morgan, helmet, mongoose, jquery bootstrap, majd létrehozom a package.json fájlt az npm init pranccsal. Ez a fájl tartalmazza a projektem legfontosabb adatait.

ng eject paranccsal legenerálom a webpack.config.js fájlt és installálom a pug template engine-t npm i pug-html-loader paranccsal, átálítom a html kiterjesztéseket pug-ra.


Ezután felépítem az alkalmazás mappa és fájl struktúrájá:

  • a weboldal aloldalainak megfelelő számban legenerálom a komponenseket: home, profile, knowledge, restfulapi, contact, messaging-service
  • index.html fájlba behúzom a bootstrapet és a jqueryt, valamint a betűtípusok beállításához a googlefonts url-jét, az ikonhasználathoz pedig a fontawesom url-jét.
  • Az app.module.ts fájlban beállítom az elérési útvonalakat a navigációhoz.
  • Az app.component.html-ben építettem fel az egyes view-k strukturáját: fix tartalomként bekerült a header, benne a navbarral és a footer, benne a social ikonokkal, a két tartalmi egység között kerülnek betöltődésre a routeroutlet tag elemek által az egyes aloldalak tartalmai.
  • Az egyes komponensek html templétjébe sectionokban helyeztem el a tartalmi egységeket igazodva a feldatkíírás tartalmi követelményeihez. A szöveges tartalmat css stílusokkal, animációkkal tettem színesebbé, változatosabbá.
  • A messaging-service.ts fájlban kerültek definiálásra a CRUD műveleteket végreható metódusok és a szükséges változók és konstansok az ES6+ szabvány szerint.
  • A messaging-service.html fájlban létrehoztam egy formot az új adatsorok felviteléhez az adatbázisba. Az adatbázisból lekért adatokat táblázatos formában az ngFor direktíva használatával jelenítettem meg.
  • Végül a regisztráció és login modalt az app.component.html-ben készítettem el, az app.component.ts-ben definiáltam a bejelentkezéshez és regisztrációhoz szükséges metódusokat.
